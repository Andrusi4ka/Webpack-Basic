Webpack Basic. Домашня робота

README

Встановлені пакети:

  1. webpack - це головний збирач проєкту, який аналізує всі файли (JavaScript, CSS, зображення, шрифти тощо), визначає залежності між ними та об’єднує все у готову, оптимізовану збірку для браузера.
  2. html-webpack-plugin - це один із найважливіших плагінів для Webpack:
    1. Створює HTML-файл у папці dist/ (під час збірки).
    2. Автоматично додає у нього всі збірки (JS, CSS, картинки), які створив Webpack.
    3. Може використовувати готовий HTML-шаблон або згенерувати новий із нуля.
    4. Підтримує мінімізацію, динамічні метадані, EJS-шаблонізацію та інше.
  3. webpack-cli - це командний інтерфейс, який дозволяє взаємодіяти з Webpack через термінал. Завдяки ньому можна запускати збірку, вибирати режими (development, production), вказувати власні конфігурації та налаштування.
  4. webpack-dev-server - це локальний сервер для розробки, який автоматично відкриває сторінку у браузері, оновлює її при зміні коду (hot reload) і дає можливість миттєво бачити результат без ручного перезавантаження чи повторної збірки.
  5. css-loader - це один із основних лоадерів Webpack, який дозволяє імпортувати CSS-файли всередину JavaScript-модулів. Без нього Webpack просто не розуміє, що робити з .css файлами.
  6. style-loader - це один з базових лоадерів для Webpack, який відповідає за вставку CSS-коду у DOM (у браузер) під час виконання. Його зазвичай використовують разом із css-loader, бо css-loader лише "зчитує" CSS, а style-loader "впроваджує" його у сторінку.
  7. sass - це сам компілятор SASS/SCSS, тобто програма, яка перетворює код з .scss або .sass у звичайний .css. Але у Webpack ми зазвичай не компілюємо вручну — для цього є sass-loader.
  8. sass-loader - це Webpack-лоадер, який підключає компілятор sass до процесу збірки. Він бере .scss файл → компілює у CSS → передає результат css-loader.
  9. html-loader - це Webpack-лоадер, який дозволяє імпортувати HTML-файли у JavaScript і обробляє всі внутрішні ресурси (зображення, посилання, скрипти тощо). Іншими словами — він навчає Webpack розуміти, що робити з .html файлами. Без нього Webpack просто не знає, як трактувати import './index.html'.
  10. PrismJS - це легка бібліотека для підсвічування синтаксису коду (syntax highlighting) у HTML-сторінках. Вона виділяє кольорами ключові слова, змінні, коментарі тощо — щоб код виглядав зрозуміліше й привабливіше.
  11. .gitignore - це спеціальний файл у Git, у якому вказується, які файли або папки не потрібно відслідковувати та додавати до репозиторію. Його використовують, щоб не заливати тимчасові файли, кеш, збірки (dist/) або залежності (node_modules/). Це допомагає тримати репозиторій чистим і зберігати тільки важливий вихідний код.
  12. @tailwindcss/postcss - це спеціальний PostCSS-плагін для TailwindCSS (v4). Його призначення — підключити Tailwind до системи PostCSS, щоб Webpack міг обробляти директиви @tailwind base, @tailwind components, @tailwind utilities. Без цього плагіна Tailwind не компілюється у CSS у сучасних версіях
  13. postcss - це сам движок PostCSS, який виконує обробку CSS-файлів перед тим, як вони потрапляють у браузер. Він дозволяє використовувати плагіни (наприклад, Tailwind або Autoprefixer), щоб автоматично додавати префікси, очищати, мінімізувати або генерувати класи. По суті, postcss — це “мозок”, який запускає всі PostCSS-плагіни у Webpack.
  14. postcss-loader - це Webpack-лоадер, який дозволяє інтегрувати PostCSS у процес збірки. Він зчитує твій postcss.config.js, підключає всі плагіни (@tailwindcss/postcss, autoprefixer) і пропускає CSS через них перед тим, як передати далі в css-loader. Без postcss-loader Webpack не зміг би “бачити” Tailwind-директиви.
  15. autoprefixer - це PostCSS-плагін, який автоматично додає вендорні префікси (-webkit-, -moz-, тощо) до CSS-властивостей.
  16. copy-webpack-plugin - це плагін для Webpack, який копіює файли або папки з вихідної директорії (наприклад src/assets/) у кінцеву директорію збірки (dist/).
  17. raw-loader - це корисний Webpack-лоадер, який використовується для імпорту вмісту файлів у вигляді звичайного тексту (string).

webpack.config.js


const path = require('path');                                  // Модуль Node.js для роботи з шляхами до файлів
const HtmlWebpackPlugin = require('html-webpack-plugin');      // Плагін, який автоматично створює HTML з підключеними скриптами
const CopyWebpackPlugin = require('copy-webpack-plugin');      // Плагін для копіювання файлів/папок у папку збірки

module.exports = {
  mode: 'development',                                         // Режим збірки: development (зручний для розробки)
  entry: './src/index.js',                                     // Головний вхідний файл (точка старту)
  output: {
    filename: '[name].[contenthash].js',                       // Ім'я файлу збірки з хешем (для кешування)
    path: path.resolve(__dirname, 'dist'),                     // Абсолютний шлях до папки з результатом збірки
    clean: true,                                               // Очищає папку dist перед новою збіркою
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/index.html',                            // HTML-шаблон, у який вставляться зібрані скрипти
      filename: 'index.html',                                  // Назва створеного HTML-файлу у dist
    }),
    new CopyWebpackPlugin({
      patterns: [                                              // Шляхи, які потрібно скопіювати під час збірки
        { from: path.resolve(__dirname, './README.md'), to: 'README.md' },  // Копіює README.md у dist
        { from: path.resolve(__dirname, 'src/images'), to: 'images' },      // Копіює папку images у dist/images
      ],
    }),
  ],
  devServer: {
    static: {
      directory: path.resolve(__dirname, 'dist'),              // Вказує папку, яку обслуговує сервер
      watch: true,                                             // Стежить за змінами у файлах dist
    },
    devMiddleware: {
      writeToDisk: true,                                       // Записує файли збірки фізично на диск
    },
    port: 8080,                                                // Порт локального сервера (http://localhost:8080)
    open: true,                                                // Автоматично відкриває сторінку в браузері
    hot: true,                                                 // Вмикає "гаряче оновлення" (без перезавантаження сторінки)
  },
  module: {
    rules: [                                                   // Набір правил для обробки різних типів файлів
      {
        test: /\.s?css$/i,                                     // Підтримка CSS і SCSS файлів
        use: ['style-loader', 'css-loader', 'postcss-loader', 'sass-loader'],
                                                               // Послідовність: вставити стилі → імпортувати → PostCSS → SCSS
      },
      {
        test: /\.(woff|woff2|eot|ttf|otf)$/i,                  // Шрифти
        type: 'asset/resource',                                // Копіює шрифти у dist і повертає їх URL
      },
      {
        test: /\.(png|jpe?g|gif|svg)$/i,                       // Зображення
        type: 'asset',                                         // Визначає: inline чи файл, залежно від розміру
        parser: { dataUrlCondition: { maxSize: 8 * 1024 } },   // Inline (base64) якщо меньше 8KB, інакше окремий файл
      },
      {
        test: /\.html$/i,                                      // HTML файли
        loader: 'html-loader',                                 // Дозволяє імпортувати HTML у JS (шаблони, компоненти)
      },
    ],
  },
};

index.js

      
import './index.html';                     // Імпортує HTML-файл у збірку Webpack (через html-loader)
import './styles/fonts.scss';              // Імпортує шрифти, оголошені у файлі fonts.scss
import './styles/style.scss';              // Основні стилі проєкту (SCSS)
import './styles/tailwind.css';            // Файл створений під Tailwind CSS

import 'prismjs';                          // Основна бібліотека Prism.js для підсвічування коду
import 'prismjs/themes/prism-okaidia.css'; // Тема оформлення Okaidia для Prism.js
import 'prismjs/components/prism-css.min.js';        // Додає підтримку підсвічування синтаксису CSS
import 'prismjs/components/prism-javascript.min.js'; // Додає підтримку підсвічування синтаксису JavaScript
